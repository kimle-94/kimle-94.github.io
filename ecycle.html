ECYCLE

WHAT IS ECYCLE?

ECYCLE is a Canadian e-bike sharing system company that provides electric bikes in many cities across Canada. The company provides multiple interconnected kiosks which allow users to rent or return e-bike based on their renting period. From now, users will not worry about returning e-bike late because the app's map will show nearby kiosks.

THE PROBLEM

Users currently have to go to kiosks to check and rent available e-bikes, which is too inconvenient and time-consuming for them. Putting it in another context, a group of friends need three e-bikes; however, they do not know how many available e-bikes at the kiosks in advance, and each station is around 15 minutes walk apart.

How night I design an app that allows people to rent e-bikes quickly and conveniently without having to worry about the availability of bikes?

THE PROPOSED SOLUTION

eCycle is an app that allows e-bikers to find nearby kiosks displayed on a map and to check if any e-bikes are available without physically being at the kiosks. In addition, users can reserve e-bikes through the app as long as there are available bikes.

BRINGING IT ALL TOGETHER

High fidelity designs with key solution

The next step was to actually build the app based on my insights, features, and goals.

1. Signing up an account and Selecting the payment method.

User can use their emails, Google accounts, Facebook accounts, or iCloud accounts to sign up and sign in. After registering successfully, the first-time user is prompted to select a payment method for renting an e-bike.

2. Going through onboarding pages, and safety rules and regulations.

The user will be guided through a four-screen onboarding process in order to highlighting the benefits and conveniences of using the app. Furthermore, the fifth onboarding will review safety rules and regulations, reminding the user to be safe on their rides.

3. Searching for nearby stations

The user want to find a station nearby in order to check available e-bikes. The user will then receive directions based on the map to get there.

4. Renting an e-bike with a 10 digit code or a QR code.

The user can either enter a 10 digit code or scan a QR code to unlock an available e-bike. Afterward, the user will select the number of bikes and the rental period. To begin the trip, click on the "Unlock" button.

It will pop up an alert card to notify the user whether they successfully rented an a-bike or not with a reminder.

5. Checking the time left and End Trip.

A pop-up card will appear telling the user how much time is left. To close the card, the user clicks on the "Cancel" button, or to end the ride, the user clicks on the "End Ride" button.

6. Reserving an e-bike with calender

User wants to reserve a ride based on their schedule. The user can go to the "Reserve a bike" screen to book an e-bike. The user must choose a specific kiosk, then select the number of e-bikes and the rental period, and select a day before clicking the "Reserve" button.

There are two options for the user to reserve a bike.
#1 The user can reserve a bike based on their particular day. #2 The user wants to reserve a bike before picking it up within an hour.

7. Checking upcoming trips or reserving an old trip.

In order to double-check their reservation, the user can go to the "My trip" page to view both their upcoming trips and their past trips. The user can reschedule or cancel an upcoming trip by deleting it or reserve an old trip.

8. Checking wallets, reminders, etc. at the profile screen.

There are six main features on the "Profile" Page. These include My Wallet to check the balance, Reminders for upcoming trips and other notifications, Customer Services, Settings, Pricing, and Terms and Conditions.

WHAT'D I DO NEXT?

Usability Testing

When the prototype design is complete, the next step is to test the target user directly to validate the design solution to the problem that I wanted to solve. However, due to time constraints, I was unable to test the app with actual users. Therefore, the next ideal steps would be to test my designs as well as usability testing to reject unnecessary features and improve user experiences.

Measure Success

If I were to develop this and deploy it to the market, I would like to verify the practicability of the app by conducting further user interviews to measure the value of my solution. Furthermore, I will implement "Dark Mode" for the app since this helps users have a better experience when using the app at night.

TAKEAWAY

Since I worked on this solo project for academic purposes, I had to conduct research from the beginning stages of identifying the issue and determining the solution to the end stages of prototype development. As a result, it was sometimes challenging for me to define the solutions and determine what features the app should or should not have.
